# ChibiCle

## 概要

ユーザの自転車走行の記録から 交差点の通行を安全に行えているか（スピードを落とせているか）を検知し、スコア化し、ユーザの親に報告する

## 背景

- 2026 年 4 月から、16 歳以上に対し、現在の自転車の罰則が厳罰化される。
- 幼いうちからの自転車の走行ルールを学ぶ必要がある。
- 検挙数が如実に高い「信号無視」・「一時不停止」に関して正しい走行ルールを教える必要がある
- 上記二つの不正に共通するのは「交差点」
- 子供が交差点を通過する際にその交差点の状況（信号や標識）を意識させることで、「信号無視」・「一時不停止」をすることを防ぐ

## 機能

### 機能 1：アカウント登録機能

- アカウントを登録する（Google）

### 機能 2：目的地設定機能

- 目的地を設定する
- 現在地からの最短経路をマップに表示

### 機能 3：走行記録機能

- 自転車の走行データを記録する
  - GPS
  - カメラ（検知時のみ？）
- 目的地についたら記録を止める

### 機能 4：走行結果照会機能

- 自転車が走行した結果を表示する
  - スコア（安全に通行できた交差点の数を何かしらスコア化したもの）
  - 写真（検知した箇所）

### 機能 5：月間データ参照機能

- 1 か月の記録をダッシュボードに表示する
  - 月間スコア（安全に通行できた交差点の数を何かしらスコア化したもの）
  - 月間走行回数

### 機能 6：過去データ参照機能

- 過去の記録をダッシュボードに表示する
  - 月間スコア（安全に通行できた交差点の数を何かしらスコア化したもの）
  - 月間走行回数
- 何かしらマップや図で表現できたら面白いなー

## 走行記録&処理&結果表示の簡単なデータフロー

1. 走行データ取得
   - フロントエンド
2. 走行データ送信
   - フロントエンド → バックエンド
3. 走行データ処理
   - バックエンド
4. 処理データ送信
   - バックエンド → フロントエンド
5. 処理データ表示

### 結果表示後過去データと月間データに登録

6. データベース登録
   - フロントエンド → データベース
7. データベース照会
   - フロントエンド
