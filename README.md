# ChibiCle

自転車走行中の交差点での一時停止を"見える化"するWebアプリ

2026年4月の自転車罰則厳罰化に先立ち、子ども・若者が正しい交通ルールを身につけるための自転車走行サポートアプリです。

---

## このアプリが解決する課題

自転車の交通違反で最も検挙数が多いのは「信号無視」と「一時不停止」です。どちらも共通するのは交差点での行動。しかし、子どもが交差点で本当に安全な走行ができているかを、保護者が毎回確認するのは現実的ではありません。

ChibiCleは、走行中のGPSデータから交差点でちゃんと速度を落とせたかを自動判定し、スコアとして可視化します。子ども自身が安全意識を持ち、保護者も走行結果を確認できる仕組みです。

---

## できること

### 1. 目的地を設定してルートを取得

出発地と目的地を地図タップまたは地名検索で設定すると、自転車用の最短ルートが表示されます。ルート上の信号のない交差点が一時停止の判定対象として自動抽出されます。

### 2. 保護者にルートを見せて確認

走行前にルート情報を保護者に提示し、安全なルートかどうかを一緒に確認できます。

### 3. 走行を記録して交差点の通行を自動判定

走行を開始すると、GPSで現在位置をリアルタイムに追跡します。交差点の15m圏内で速度3.0km/h未満まで減速していれば「一時停止できた」と自動判定されます。経路を外れた場合は自動的にリルートされます。

### 4. 走行結果をスコアで確認

走行終了後、安全に通行できた交差点の数がスコアとして表示されます。一時停止できなかった交差点の位置も地図上で確認できるため、次回の改善につなげられます。

### 5. 月間データで成長を実感

月ごとのスコアや走行回数をダッシュボードで確認でき、安全運転の習慣化を促します。

---

## 利用の流れ

1. 目的地を設定 — 地図タップや検索で出発地・目的地を選択
2. ルートを確認 — 保護者と一緒にルートを確認
3. 走行開始 — GPSで自動追跡し、交差点の減速をリアルタイム判定
4. 結果を確認 — スコアと地図で安全度を確認

---

## 判定のしくみ

判定のための閾値は、backend-go/config/config.goで編集できます。

| 項目 | 内容 |
|------|------|
| 判定対象 | ルート上の信号のない交差点（T字路以上） |
| 一時停止の基準 | 交差点15m圏内で速度 3.0 km/h 未満 |
| 経路逸脱 | ルートから 50m以上 離れると自動リルート |
| GPS精度フィルタ | 精度20m超のデータは判定に使用しない |

## 技術構成

| レイヤー | 技術 |
|---------|------|
| フロントエンド | React 19 + TypeScript, Vite, Tailwind CSS, Leaflet（地図） |
| 状態管理 | Jotai（リアルタイムUI）, Dexie/IndexedDB（オフラインデータ） |
| バックエンド | Go（Clean Architecture）, go-chi |
| 認証・DB | Supabase（Google認証, 月間データ保存） |
| 外部API | OSRM（自転車ルーティング）, Overpass（交差点・公道判定）, Photon（地名検索） |

### アーキテクチャ概要

- フロントエンド（PWA） — GPS取得、IndexedDBへのバッファリング、5秒ごとのバッチ送信、地図表示（Leaflet）、認証（Supabase）
  - ↕ REST API
- Goバックエンド — ルート計画、交差点抽出、一時停止判定、リルート
  - → OSRM — 自転車ルーティング
  - → Overpass — 公道判定・信号フィルタ
  - → Photon — 地名検索

## セットアップ

### フロントエンド

```bash
cd frontend
npm install
npm run dev
```

### バックエンド

```bash
cd backend-go
go run ./cmd/server
```

環境変数はプレフィックス `BTD_` で設定可能です（例: `BTD_PORT`, `BTD_OSRM_BASE_URL`）。デフォルト値が設定されているため、開発環境ではそのまま起動できます。
